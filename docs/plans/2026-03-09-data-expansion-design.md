# claude-stats Data Expansion Design

**Date:** 2026-03-09
**Status:** Approved
**Scope:** Subagent tracking, task management, project memories, file workflow, error tracking, telemetry, security warnings

## Features

### 1. New "Agents" Tab (Main Dashboard)

**Subagent Overview:**
- Donut chart of subagent types (Explore, Plan, code-reviewer, general-purpose etc.)
- Bar chart: Top-10 most frequent subagent descriptions
- Sortable table: Session-ID, Type, Description, Parent-Session, Tokens, Cost

**Task Overview:**
- Total tasks, completion rate as donut/gauge
- Status distribution (pending/in_progress/completed)

**Plan-Mode Plans:**
- List of `.md` plans from `~/.claude/plans/` with title, created date, size
- Clickable — opens rendered plan page

**Error Overview:**
- Error rate per tool as horizontal bar chart (Errors / Total Calls)
- Top error messages

**Data pipeline:** `parse_session_transcripts()` extracts from each `tool_use` with `name: "Agent"` the `subagent_type` and `description`. From `tool_result` with `is_error: true` the error counters. Subagent JSONL files (already loaded via `rglob("*.jsonl")`) are parsed and associated with parent sessions. Tasks from `~/.claude/tasks/`. Plans from `~/.claude/plans/`.

### 2. Extended Project Pages

**Tab "Overview"** (existing content + extensions):
- Existing stats cards and session list
- **Memory Card:** Rendered markdown content of project's `MEMORY.md` (if present), with collapse/expand. Opt-out via `--no-memories` flag.
- **Subagent Usage:** Compact overview of subagent types used in this project (tags with counters)
- **Top Files:** Top-10 most edited files (Read/Edit/Write counts from tool calls)
- **Error Rate:** Error rate for this project
- **Git Ops:** Count of commits, PRs, pushes by Claude in this project

**Tab "Workflow"** (new):
- Vertical timeline, chronological, one block per session
  - Session start as marker with date/time
  - File operations as color-coded entries (Read=blue, Edit=cyan, Write=green, Bash/git=orange)
  - Commit messages as milestone markers (highlighted)
  - Subagent dispatches as indented sub-blocks
  - Compaction events as warning markers
- Filterable by file type, operation, time range
- Virtual scrolling for performance with large projects

**Data extraction:** For each project session, file paths + timestamps + operations extracted from tool calls. Git commits from Bash tool calls containing `git commit`.

### 3. Telemetry & Performance (Insights Tab)

**System Info Card** (static):
- OS, Shell, Node version, Platform — from newest shell snapshot
- Claude Code version (from latest debug log)
- Configured MCP servers (from `.claude.json`)

**Performance Metrics** (time series):
- Memory usage per session: line/bar chart — peak memory from telemetry events
- Session duration vs. token usage: scatter plot — shows proportionality or outliers
- Error rate over time: line — daily error rate (Errors / Total Tool Calls)

**Error Detail Card:**
- Top-10 most frequent error messages with count
- Error rate per tool as compact table

**Data extraction:** Telemetry JSONs from `~/.claude/telemetry/` — `process.memoryUsage`, `process.uptime`, `process.cpuUsage` per event. Errors from `is_error: true` tool results in session transcripts.

### 4. Security & Privacy

**README warning box:**
Prominent red warning at the top of README: "This dashboard may contain sensitive data (source code, file paths, API keys, memories, conversation history). Do NOT publish to public internet or unsecured locations."

**CLI flags:**
- `--no-memories` — Skip embedding memories in project pages
- Existing `--skip-sessions` — Skip session replay pages
- Console warning on generation: "Output contains sensitive data"

**Dashboard footer:**
Small hint: "Contains private data — do not share publicly"

## Technical Decisions

- No new frameworks — stays Vanilla JS/HTML/CSS, generated by Python
- Subagent JSONL files already loaded by `rglob("*.jsonl")` — just need identification and parent association
- Telemetry parsing is new I/O but files are small (~22 MB total)
- Project page tabs via CSS/JS tab switching (no separate HTML files per tab)
- Workflow timeline uses virtual scrolling for large projects
- All new locale keys added to en.json and de.json
- Version bump to 0.4.0
